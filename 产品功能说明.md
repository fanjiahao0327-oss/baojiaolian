# AI 保险教练 — 完整功能逻辑文档

## 一、产品定位

面向 **保险代理人** 的 AI 策略参谋工具。代理人将面谈中遇到的客户情况填入 KYC 表单，AI 基于一套完整的顾问式销售方法论，给出 **卡点诊断 → 推演方向 → 建议话术** 三段式指导。

## 二、技术架构

| 层 | 技术 |
|----|------|
| 前端 | Next.js 16 + React 19 + TypeScript + Tailwind CSS 4 |
| 后端 | Next.js API Route Handlers（同仓库，无需额外服务器） |
| AI | DeepSeek API（通过 openai 兼容 SDK 调用），流式输出 |
| 数据库 | SQLite（better-sqlite3），文件存储在 data/app.db |
| 认证 | iron-session（cookie-based），手机号 + 验证码登录 |
| 部署 | 单机 Node.js 服务，npm run build && npm run start |

## 三、用户操作流程

### 1. 登录
- 输入手机号 + 验证码（开发阶段固定 1234）
- 新用户自动注册并赠送 **10 积分**
- 每次 AI 对话消耗 **1 积分**，积分不足时弹窗提示前往充值页
- 积分页可手动充值（开发阶段直接加积分，无需支付）

### 2. 填写 KYC 表单（6 步向导）
代理人在表单中录入当前面谈客户的信息，分为 6 个步骤。

### 3. AI 教练诊断
点击"提交给保险教练诊断"后：
1. 前端将表单数据拼接成结构化文本（含客户画像、财务、保障、面谈入口、卡点）
2. 发送给 POST /api/coach，API 将知识库（~8000 字顾问式销售方法论）作为 System Prompt
3. AI 通过流式输出（SSE）实时显示三段式诊断结果
4. 对话自动保存到历史记录

### 4. 追问
诊断完成后，可在输入框追问（补充客户信息、反馈面谈结果等），AI 结合历史上下文继续给出建议。

### 5. 历史对话
可通过侧边栏查看、加载历史诊断对话。

### 6. 面谈复盘（/review 页面）
独立于 KYC 向导的复盘工具，每次面谈后记录：基本信息复盘、需求状态、动机原话、关键问题、回顾失误、社交激活检查、整体总结。

## 五、AI 系统设计

### 5.1 调用链路

```
前端 page.tsx → fetch("/api/coach") → 拼接 System Prompt
  → DeepSeek API (deepseek-v4-pro, stream: true, temperature: 0.7)
  → 流式输出 → 前端实时渲染 Markdown
```

### 5.2 System Prompt 结构

```
# 角色
你是「AI 保险教练」—— 一名专为保险代理人提供幕后策略指导的顶级专家。

# 工作原则
1. 绝对尊重客户 — 永远不批评客户的选择
2. 积极导向提问 — 用积极假设打开话题
3. 挖掘深层需求 — 看穿表面陈述，找到真正的担心或期待
4. 客户利益优先 — 一切建议从客户真实需求出发

# 输出结构（三段式）
## 一、卡点诊断
- 2-3 句话点明核心卡点
- 卡点的本质（信任问题？需求不明确？异议未化解？流程卡壳？）
- 代理人在当下环节可能犯的典型错误

## 二、推演方向
- 应走哪个方向（谈保障？教育金？养老？增员？）
- 选择依据
- 下一步该"敲"哪几堵墙，按优先级排列
- 1-2 个具体的积极导向提问话术

## 三、建议话术
- 2-3 段可直接使用的话术模板
- 话术须符合"积极假设 + 具体指向 + 带问号"结构
- 标注每段话术的适用场景和注意事项

# 风格要求
- Markdown 格式，善用加粗、列表、引用
- 语言干练、直击要点
- 建议"拿来就能用"，避免纯理论

# 销售方法论知识库
{约 8000 字顾问式销售方法论}
```

### 5.3 知识库涵盖的 8 大模块

| 模块 | 核心内容 |
|------|----------|
| 认知转型 | 强关系→弱关系、50/60后→80/90后客户群变化、产品推销→顾问式销售的"卖水"模型 |
| 客户定位 | 五级金字塔（贫困→小中产→典型中产→富裕中产→高净值），每层画像/需求/策略；五大普世需求缺口（工作收入/身体疾病/孩子教育/退休养老/财富传承） |
| 社交激活 | 社交线索原则、社会交换原则（有用/有趣/有共鸣）、微信资源梳理、朋友圈激活法、公司活动策略 |
| 线上转面谈 | 电话邀约四步：加热关系→告知好处→化解阻力→二选一时间 |
| 面谈三步流程 | 第一步找需求（主动咨询4场景 + 非主动咨询"拆墙法"分析案例）、第二步强化需求（保障类谈后果+内生保额 / 储蓄类畅想未来+三角形预期管理）、第三步找不足给方案（5 个典型场景：存不下钱/有积蓄/做投资/卖房子/传承） |
| 储蓄险四风险 | 投资风险、挪用风险、利率下行风险、使用期风险（含晚年消费/亲戚借用/长寿风险/子女考验/护理缺口） |
| 异议处理 | 三原则（防御不是异议 / 先甄别真假 / 动机再现）、六大高频异议逐个击破（跟家人商量/手头紧/比较产品/通胀贬值/交不上保费/保费倒挂） |
| 复盘体系 | 每次面谈后必做的复盘模板：基本信息→需求状态→动机原话→关键问题→回顾失误→社交激活检查 |

```
- 性别：男
- 所在城市：上海
- 婚姻状况：已婚
- 子女数量：2

## 职业与经济基础

- 配偶职业-行业：金融
- 配偶职业-职位：风控经理
- 职业状态感知：稳定或上升期

## 家庭财务感知
- 主要收入来源：工资收入 / 投资分红
- 资产感知：有自住房 / 有基金或股票等投资

- 医疗险：有，百万医疗，年交500元
- 重疾险：有，50万保额，终身，年交8000元

- 触发场景：客户主动咨询（有明确原话）

- 客户提出的异议或卡点原话：我觉得你们产品收益不如基金
- 代理人当时的回应方式简述：我解释了保险的安全性

请给出：1. 卡点诊断 2. 推演方向 3. 建议话术
```

## 七、配套功能

### 7.1 积分系统
- 初始 10 分（新用户注册时自动赠送）
- 每次 AI 对话消耗 1 分（在流式输出完成后扣减）
- 积分不足时 API 返回 402，前端弹窗提示充值
- 积分页 `/points`：显示余额 + 充值输入框 + 积分流水列表

### 7.2 面谈复盘 `/review`
- 独立页面，不依赖 KYC 向导
- 字段：基本信息复盘、需求状态（主动/非主动）、动机原话、关键问题、回顾失误、社交激活检查、整体总结
- 支持新建、查看详情、列表浏览

### 7.3 移动端适配
- 桌面端：左右分栏（左侧 KYC 表单 + 右侧教练面板）
- 移动端：顶部标签切换（信息填写 / 教练指导）
- 表单区、教练区、历史对话面板均做了响应式处理

## 八、数据库表结构

```sql
-- 用户表
users (id, phone UNIQUE, created_at, last_login_at)

-- 积分流水表
point_transactions (id, user_id FK, amount, type CHECK(charge/consume), description, created_at)

-- 面谈复盘表
```

## 九、待讨论的优化方向

1. **KYC 表单字段** — 当前 6 步 30 字段，是否覆盖全面？是否缺少关键信息维度（如客户性格类型、过往购买行为）？
2. **知识库长度** — 8000 字方法论是否太长（每次消耗大量 token）？是否需要按场景动态裁剪？
3. **输出结构** — 三段式是否够用？是否需要增加"面谈模拟/角色扮演"或"异议预演"功能？
4. **客户档案管理** — 当前每次诊断独立，是否需要长期追踪同一客户的多轮面谈进展？
5. **积分商业化** — 当前手动充值，未来是否对接微信支付？定价策略是什么？
6. **险种覆盖** — 当前以重疾险/储蓄险为主，是否需要扩展医疗险、意外险、团险等品类的专项方法论？
7. **多模型支持** — 是否需要支持切换不同 AI 模型？
