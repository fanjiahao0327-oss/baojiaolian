# AI 保险教练 — 完整优化实施计划

## 总览

共 **13 项**优化，分为 5 批实施，预计改动约 20 个文件。

## 第一批：数据基础（客户档案系统）

### 1-1. 新建 clients 表

**文件：** `src/lib/db.ts`

```sql
CREATE TABLE IF NOT EXISTS clients (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  name TEXT NOT NULL DEFAULT '未命名客户',
  kyc_snapshot TEXT NOT NULL DEFAULT '{}',
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 1-2. conversations 表增加 client_id 字段

**文件：** `src/lib/db.ts`

加迁移逻辑：检查 client_id 列是否存在，不存在则 ALTER TABLE 添加。

### 1-3. 客户管理 API

新增 3 个接口：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/clients` | 查该代理人的所有客户，返回 id、name、updated_at，按更新时间降序 |
| POST | `/api/clients` | 创建新客户，body: `{ name?, kycSnapshot? }` |
| PUT | `/api/clients/[id]` | 更新客户 name 或 kycSnapshot，自动刷新 updated_at |

### 1-4. 改造 POST /api/coach

- 接收参数新增可选 `clientId`
- 诊断成功后：将 conversation 关联 client_id，更新 client 的 kyc_snapshot
- 若未传 clientId，自动创建新客户（name 从 KYC 数据生成，如"35岁男-上海"）

### 1-5. 前端客户管理组件

**文件：** `src/components/KYCWizard.tsx`、`src/app/page.tsx`

- KYC 第 1 步上方增加客户选择下拉框（已有客户列表 + "新建客户"）
- 选已有客户 → 回填 kyc_snapshot 到表单
- 选新建客户 → 清空表单
- 提交诊断时传 clientId
- 历史对话页展示关联客户名称，点击查看该客户全部诊断
- 支持编辑客户名称（inline 编辑或弹窗）

## 第三批：AI 能力增强

### 3-1. 知识库模块化

**文件：** `src/lib/knowledge.ts`（改造为模块数组）

每个模块结构：
```ts
{
  id: string;
  category: string;
  tags: string[];
  content: string;
  estimatedTokens: number;
}
```

**拆分类别（至少包含）：**

| 类别 | 模块数 | 说明 |
|------|--------|------|
| 基础模块 | 1 | 角色、三原则、输出结构、风格要求（始终加载） |
| 认知转型 | 1 | 三大变化 + 销售模式转变 |
| 客户金字塔 | 5 | 贫困层、小中产、典型中产、富裕中产、高净值 |
| 社交激活 | 1 | 微信资源梳理 + 朋友圈激活 |
| 线上转面谈 | 1 | 电话邀约话术 |
| 面谈流程-主动咨询 | 2 | 找需求（4场景）+ 强化需求 + 找不足 |
| 面谈流程-非主动咨询 | 1 | 拆墙法 + 案例分析 |
| 储蓄险风险 | 1 | 四大风险精讲 |
| 异议处理 | 6 | 每个高频异议一个模块 |
| 复盘体系 | 1 | 复盘模板 |
| **总计** | **20 个模块** | |

### 3-2. 模块选择函数

**文件：** `src/lib/module-selector.ts`（新建）

```ts
function selectModules(kycData: KYCFormData): string[]
```

**选择逻辑：**

1. 始终加载基础模块
2. `triggerScenario` 含"主动" → 加载主动咨询面谈流程；含"非主动" → 加载非主动咨询流程
3. 根据 `annualIncome` + `stockAmount` + `savingsAmount` + `assets` → 判断客户层级，只加载对应金字塔模块
4. `clientObjection` 非空 → 按关键词匹配对应异议模块（如含"商量"匹配"跟家人商量"模块）
5. `childrenDetail` 非空 + `childrenCount > 0` + 原话含"教育/出国" → 加载教育金相关
6. `expensePressure` / `incomeSources` / `careerStatus` → 辅助判断
7. 去重，控制在 2500-4000 token

### 3-3. 改造诊断 API

**文件：** `src/app/api/coach/route.ts`

- 用 `selectModules()` 替代硬编码 `COACH_KNOWLEDGE_BASE`
- 诊断成功后，将本次使用的模块 ID 列表存入 conversation 的 messages 中
- 追问时从历史消息中取出模块 ID，复用相同组合

### 3-4. 面谈事前准备模式

**文件：** `src/app/page.tsx`、`src/components/CoachPanel.tsx`

- 在教练面板增加模式切换按钮：诊断模式 / 准备模式
- 准备模式下，System Prompt 调整为：
  - 重点输出：客户分析、推荐敲墙方向（按优先级）、预判可能的异议及预案、建议的社交激活话题
  - 不用输出"卡点诊断"（因为还没面谈）
- 准备模式不扣积分（或扣 0.5 分）

### 3-5. 引导式追问选项

**文件：** `src/app/api/coach/route.ts`、`src/components/CoachPanel.tsx`

- AI 每段回复末尾，增加输出 2-3 个引导式追问
- 格式：`---\n💡 进一步讨论：\n- [追问1]\n- [追问2]\n- [追问3]`
- 前端解析，渲染为可点击按钮，点击自动填入追问输入框

## 第五批：健壮性

### 5-1. 积分保护 — 数据库事务

**文件：** `src/app/api/coach/route.ts`

当前扣分代码：
```ts
db.prepare("INSERT INTO point_transactions ...").run(...)
```

改造为事务：
```ts
const txn = db.transaction(() => {
  const row = db.prepare("SELECT COALESCE(SUM(amount), 0) as balance FROM point_transactions WHERE user_id = ?").get(userId);
  if (row.balance < COST_PER_CALL) throw new Error("积分不足");
  db.prepare("INSERT INTO point_transactions (user_id, amount, type, description) VALUES (?, ?, 'consume', ?)").run(userId, -COST_PER_CALL, "对话消耗");
});
txn();
```

### 5-2. 错误处理白盒化

**文件：** 全局替换所有 `} catch { // ignore }` 为空 catch 的地方

- 每个 catch 至少加 `console.error("模块名", error)`
- 涉及文件：`UserProvider.tsx`、`ConversationList.tsx`、`page.tsx`、`points/page.tsx`、`review/page.tsx`
- 约 10 处需要修改
